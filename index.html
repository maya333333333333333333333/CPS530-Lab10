My Materialize Page
Problem 2 – Framework Write-Up
1. Framework Chosen

I chose MaterializeCSS, a lightweight front-end CSS framework based on Google's Material Design guidelines.

2. How I Installed It

I used the CDN version. I simply added the MaterializeCSS stylesheet and script into the head of my HTML file. No installation or dependency manager was needed.

3. How I Built the Page

I created a simple HTML page and used Materialize components such as a navbar, container, and card. I also used Google Fonts to make the page look more modern.

4. Difficulties Encountered

At first, the Materialize JavaScript components were not working because I forgot to include the Materialize JS file and the script was in the wrong order. After placing the script tag under the CSS link, everything started working.

I also had trouble hosting the page at first because GitHub Pages only serves from a special branch or /docs folder. I solved this by creating a new repository, enabling GitHub Pages inside Settings → Pages, and placing my HTML file in the root of the repository.
