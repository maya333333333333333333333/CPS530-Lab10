Problem 2

For this assignment, I chose the front-end framework MaterializeCSS because it is lightweight, easy to use, and works directly through a CDN without requiring a backend server or build process. To install it, I simply created an index.html file and added the CDN link to MaterializeCSS in the header. This allowed me to immediately use the framework's typography, grid system, and simple styling elements.

To build this page, I designed a header section using a custom background color and a Google Font. Below it, I added a centered content block with normal paragraphs explaining the installation process, build steps, and challenges. GitHub Pages was used to host the site: I created a public repository, uploaded the file, and enabled GitHub Pages under the repository settings so the page became publicly accessible.

During the process, the main difficulty I encountered was getting GitHub Pages to serve the site correctly. At first, I received a 404 error because my file was not placed in the root directory and the Pages branch settings were incorrect. After ensuring index.html was in the repository root and confirming that GitHub Pages was set to deploy from the main branch, the site loaded successfully. Another small issue was ensuring that the MaterializeCSS CDN link loaded over HTTPS, as GitHub Pages requires secure connections, but this was solved by using the official CDN link.

Overall, the experience provided a simple introduction to using a front-end CSS framework and deploying a small project using GitHub Pages.
